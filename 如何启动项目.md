# 如何启动项目

## ⚠️ 重要提示

**这个项目不能直接用 Live Server 打开！**

这是一个使用 Vite + React + TypeScript 的项目，需要：
1. 安装 Node.js 环境
2. 安装项目依赖
3. 使用 Vite 开发服务器启动

Live Server 只是一个简单的静态文件服务器，无法处理：
- TypeScript 文件 (.tsx)
- JSX 语法
- 模块导入 (import)
- React 组件
- Tailwind CSS 编译

## 正确的启动步骤

### 第一步：安装 Node.js（如果还没有安装）

1. 访问 [nodejs.org](https://nodejs.org/)
2. 下载并安装 LTS 版本（推荐 18.x 或更高）
3. 安装完成后，打开终端验证：
```bash
node --version
npm --version
```

### 第二步：安装项目依赖

在项目目录中打开终端，运行：

```bash
npm install
```

或者如果使用 pnpm：

```bash
pnpm install
```

这会安装所有需要的依赖包（包括 React、Vite 等）。

### 第三步：启动开发服务器

安装完成后，运行：

```bash
npm run dev
```

或者：

```bash
pnpm dev
```

Vite 会启动开发服务器，通常会显示类似这样的信息：

```
  VITE v6.x.x  ready in xxx ms

  ➜  Local:   http://localhost:5173/
  ➜  Network: use --host to expose
```

### 第四步：在浏览器中打开

在浏览器中访问显示的地址（通常是 `http://localhost:5173/`）

## 常见问题

### Q: 为什么 Live Server 打开后是空白页面？

A: Live Server 无法处理 TypeScript 和 React。浏览器控制台可能会显示错误，比如：
- "Failed to load module script"
- "Unexpected token '<'"
- "Cannot use import statement outside a module"

### Q: 如何检查是否正确安装了 Node.js？

在终端运行：
```bash
node --version
npm --version
```

如果显示版本号，说明已安装成功。

### Q: npm install 很慢怎么办？

可以：
1. 使用国内镜像源：
```bash
npm config set registry https://registry.npmmirror.com
```

2. 或使用 pnpm（更快）：
```bash
npm install -g pnpm
pnpm install
```

### Q: 启动后浏览器还是空白？

1. 检查终端是否有错误信息
2. 打开浏览器开发者工具（F12），查看 Console 标签页的错误
3. 确认是否访问了正确的地址（通常是 localhost:5173）

## 开发时的工作流程

1. 修改代码
2. 保存文件
3. Vite 会自动重新编译（热更新）
4. 浏览器自动刷新显示最新内容

## 生产构建

如果要构建生产版本：

```bash
npm run build
```

构建后的文件会在 `dist` 目录中，这些文件可以部署到任何静态文件服务器（包括 Vercel）。
